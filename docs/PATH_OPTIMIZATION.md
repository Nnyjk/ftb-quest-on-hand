# 方向演进记录

> 本文档记录了项目从 v1 到 v2 的思路转变过程
> 保留作为历史参考，不影响当前设计

---

## v1 方向：FTB Quest + 工作台侧边栏

**核心思路**：在工作台 UI 旁添加侧边栏，展示任务目标物品和进度。

**发现的问题**：JEI 书签 + FTB 任务书已经能较好地解决这个场景，差异化不够。

## v2 方向（当前）：Plan Board

**核心思路**：监听 JEI 书签 → 同步到计划板 → 手持计划板右键容器自动提取缺失物品。

**转变动机**：
- JEI 书签已经是玩家的习惯性操作
- 真正的痛点是"配方和容器之间缺乏连接"
- 计划板作为"物流自动化"工具，差异化清晰
