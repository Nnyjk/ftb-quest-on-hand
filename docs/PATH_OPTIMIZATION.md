# FTB Quest On Hand — Quest + JEI 路径体验优化

> 当前路径痛点分析 + 优化方案
> 目标读者：开发者（你我）
> 基于 2025-05-25 讨论修订

---

## 1. 当前标准路径

```
任务书（FTB Quests）
    → 逐个点击任务目标物品添加到 JEI 书签
    → 关闭任务书，打开 JEI
    → JEI 书签栏查找物品
    → JEI 快捷合成（Shift+点击）
```

这条路径有 **4 个主要断点**：

| # | 断点 | 问题描述 |
|---|------|---------|
| ① | 逐个添加书签 | 一个任务可能有 5~10 个目标物品，必须一个个手动添加到 JEI书签，操作繁琐 |
| ② | 书签无上下文 | JEI 书签列表是扁平的，5个任务 × 10个物品 = 50个书签混杂在一起，哪个是哪任务的？ |
| ③ | 进度不联动 | JEI 不知道你还需要多少物品，合成了多少，任务进度多少 |
| ④ | 路径太长 | 至少需要"任务书 → JEI → 快捷合成"三次界面切换 |

---

## 2. 优化方案

### 2.1 一键批量添加（解决断点①）

**原流程**：逐个点击任务目标物品 → 添加到 JEI 书签

**优化流程**：
- 任务书界面中，在任务标题旁新增 **"批量添加到 JEI"** 按钮
- 点击后弹出确认浮窗，显示"将添加 N 个目标物品到 JEI 书签"
- 可选：自动去重（已书签中的物品不再重复添加）

**实现方式**：
- 在 FTB Quests 任务书 Screen 中注入一个按钮
- 遍历该任务的 `ItemObjective`，批量调用 JEI `BookmarkPlugin.addBookmark(items)`

**效果**：原来 10 个物品需要 10 次点击 → 优化后 1 次点击。

---

### 2.2 书签上下文标注（解决断点②）

**原流程**：JEI 书签栏 → 扁平的物品列表

**优化流程**：
- JEI 书签栏中，每个书签条目显示 **来源任务图标 + 任务名**
- Hover 时显示完整任务名 + "任务目标：X/Y 已完成"
- 提供 **过滤按钮**：一键只看"来自任务书的书签"

**实现方式**：
- JEI 的书签数据结构支持附加 `String` tooltip（显示在物品名下方）
- 写入时附加上 `QuestID → 任务名` 的映射字符串
- 通过 `IGuiBookmarkList.addEntry()` 时附加额外 tooltip

**效果**：50个书签不再是杂乱的列表，而是按任务分组、有上下文的集合。

---

### 2.3 任务进度联动（解决断点③）

**原流程**：JEI 书签不知道任务进度，合成完也不知道还差多少

**优化流程**：
- Hover 书签物品时，显示 **"任务进度：还需要 X 个"**（从 FTB Quests API 读取 `current - target`）
- 侧边栏或任务书界面，显示当前任务的**目标物品完成进度条**

**这是一个可选功能**，因为：
- FTB Quests 的进度数据可能需要频繁轮询
- 但对于玩家来说，"我合成了2个钻石锭，任务还差1个"这个信息极有价值

**效果**：JEI 书签从"物品列表"升级为"任务进度看板"。

---

### 2.4 工作台侧边栏：任务进度实时视图（解决断点④）

**核心思路**：工作台旁边的侧边栏，显示**当前聚焦任务**的目标物品清单 + 实时进度

```
┌─────────────────────────┬──────────────────┐
│                         │ [任务：钻石剑] ▼   │  ← 当前聚焦任务下拉选择
│                         │                    │
│      原版工作台          │  ◆钻石锭 2/3  ✓   │  ← 已完成的目标（划线/变灰）
│       3×3 网格          │  ◇木棍   1/2  ○   │  ← 未完成的目标（高亮）
│                         │  ◇铁锭   0/1  ○   │
│                         │                    │
│                         │  [同步到JEI书签]   │  ← 一键批量添加
└─────────────────────────┴──────────────────┘
```

**与断点④的关联**：
- 原来需要"任务书 → JEI → 工作台"三次切换
- 优化后：打开工作台 → 侧边栏就是当前任务清单 → 直接合成 → 进度实时更新
- 任务书变成了"设置聚焦任务"的场所，而不是合成时的参照物

**侧边栏功能定位重新定义**：

| 版本 | 侧边栏定位 |
|------|-----------|
| v1（原始） | 收藏栏（类似个人仓库） |
| v2（修订） | 任务目标进度面板（当前聚焦任务的实时清单） |

## 3. 修订后的功能范围

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **F1. 批量添加 JEI 书签** | 任务书界面一键添加所有目标物品到 JEI 书签 | P0 |
| **F2. 书签上下文标注** | JEI 书签显示来源任务名/图标 | P0 |
| **F3. 任务进度联动** | Hover 书签显示"还需X个"；侧边栏显示进度条 | P1 |
| **F4. 工作台侧边栏（修订）** | 聚焦任务的实时目标清单 + 进度，不做收藏 | P1 |
| **F5. 外部 mod API** | AE2/RS 等接入（保留，扩展进度查询能力） | P2 |

## 4. 修订后的侧边栏 UI 规格

### 4.1 布局

```
┌────────────────────────────────┬─────────────────────┐
│                                │ [▼ 钻石剑任务    ]   │  ← 任务下拉选择器
│      原版工作台                 │                      │
│       3×3 网格                  │  ◇ 钻石锭   2/3  ●  │  ← ●=未完成
│                                │  ◇ 木棍     1/2  ●  │
│                                │  ◆ 铁锭     3/3  ✓  │  ← ✓=已完成
│                                │                      │
│                                │  [⚡ 同步 JEI 书签]   │
└────────────────────────────────┴─────────────────────┘
```

### 4.2 任务选择器

- 点击下拉 → 显示玩家已接取的所有 FTB 任务列表
- 搜索框支持模糊过滤
- 选择任务后，侧边栏自动刷新为该任务的目标物品清单

### 4.3 目标物品行

- **未完成**：高亮，正常显示数量 `current/target`
- **已完成**：变灰 + 划线，Tooltip 显示"已完成"
- **数量来源**：实时从 FTB Quests API 读取当前玩家进度

### 4.4 同步按钮

- 点击后，将该任务**所有未添加到书签**的目标物品批量添加到 JEI 书签
- 已在书签中的物品自动跳过（去重）

## 5. 修订后的架构调整

### 5.1 核心层变更

`BookmarkManager` 职责变更：

```java
// 不再是"收藏管理器"，而是"任务上下文管理器"
public class QuestContextManager {
    // 当前聚焦的任务
    public static void setFocusedQuest(String playerId, String questId);
    public static String getFocusedQuest(String playerId);

    // 聚焦任务的目标物品进度（实时）
    public static List<QuestObjectiveProgress> getFocusedObjectives(String playerId);

    // JEI 书签操作（仍然保留）
    public static void addToJei(String playerId, ItemStack item);
}
```

### 5.2 FTB Quest 集成变更

```java
public interface QuestSync {
    // 读取任务目标物品（非奖励）
    List<ItemStack> getQuestObjectives(String questId);

    // 读取玩家当前任务目标进度（current / target）
    Map<String, Integer> getObjectiveProgress(String questId, String playerId);

    // 批量添加目标物品到 JEI 书签
    default void batchAddToJei(String questId, String playerId) {
        List<ItemStack> items = getQuestObjectives(questId);
        for (ItemStack item : items) {
            BookmarkManager.addToJei(playerId, item);
        }
    }
}
```

## 6. 目标用户体验对比

| 维度 | 优化前 | 优化后 |
|------|--------|--------|
| 添加书签 | 逐个点击 N 次 | 1 次 |
| 查看任务物品 | 切换到任务书 | 工作台侧边栏直接看 |
| 了解完成进度 | 打开任务书核对 | 侧边栏实时显示 |
| 界面切换次数 | 任务书→JEI→工作台（3次） | 工作台（1次，含侧边栏） |
| 书签管理 | 扁平混乱 | 按任务分组+标注 |
